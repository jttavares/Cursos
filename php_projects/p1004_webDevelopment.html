Desenvolvimento Web

Os termos front-end e back-end referem-se às áreas de atuação comuns dentro do desenvolvimento web. Claro que existem outras “divisões” dentro da área, mas para fins didáticos e com o objetivo de esclarecer as diferenças entre front-end e back-end, esses dois serão detalhados.

Front-end e back-end são termos generalizados que se referem às etapas inicial e final de um processo. O front-end é responsável por coletar, em várias formas, a entrada do usuário e processá-la para adequá-la a uma especificação útil para o back-end. O front-end é uma espécie de interface entre o usuário e o back-end.

A explicação é simples e, de certa forma, tecnicamente não muito precisa, porém adequada para o entendimento inicial sobre a diferença entre front-end e back-end. Então, para melhor compreensão dos conceitos (ainda sem aprofundamento, mas de forma a deixá-los mais claros), acompanhe breve explicação sobre cada um deles.

Front-End

Definição curta: Front-end cuida da interface e da experiência do usuário.

Front-end é responsável por coletar, em várias formas, a entrada do usuário e processá-la para adequá-la a uma especificação útil para o back-end. Porém, a coleta de informações não é necessariamente obrigatória para processamento no back-end. Podem existir (e, efetivamente, há) sites que não demandam do recolhimento de quaisquer informações, tais como os sites de exibição de vídeos ou de notícias.

Na verdade, o front-end é responsável por desenvolver a interface em que as pessoas que acessam o site, software, sistema ou quaisquer formas de interação online, terão acesso.

São os profissionais de front-end que fazem com que o conteúdo (ou as informações do site) seja apresentado de maneira tecnicamente adequada. São eles que preparam a organização de sua estrutura e planejam como será a organização do conteúdo; que apresentam a informação de maneira agradável e de forma que atenda aos objetivos do projeto, através do web design; e que planejam a experiência do usuário, tentando fazer com que seja a melhor possível.

Em suma, o profissional de front-end deve se preocupar com o visitante do site: em como ele verá as informações, como conseguirá encontrar o que procura e como interagirá com os recursos, tudo da maneira mais simples e eficiente possível.

Algumas das tecnologias e áreas de estudo que podem ser consideradas pertencentes ao front-end são:

  • HTML
  • CSS
  • JavaScript
  • Web Design
  • Design de Interação
  • Arquitetura da Informação
  • Estratégia de Conteúdo

Estas são algumas das perguntas que cabe ao profissional do front-end responder, e ser tecnicamente capaz de solucionar/desenvolver:

  • “O site está bem estruturado e será bem indexado pelo Google?”
  • “O site vai ficar “bonito” e funcional?”
  • “Como serão as interações no site?”
  • “O site funcionará em qualquer dispositivo (desktop, tablet, smartphone etc.) e sistema operacional (Windows, Linux, Mac OS etc.)?”
  • “As informações podem ser encontradas facilmente?”
  • “O conteúdo (texto, imagem, vídeo etc.) apresentado em cada seção é o mais adequado?”
  • “As páginas vão demorar a carregar?”

Com essas pequenas explicações e exemplos já é possível perceber que trabalhar no front-end não é tarefa simples. Como explicado, não necessariamente o profissional da área precisa dominar todas as tecnologias citadas (e foram somente algumas) e nem responder a todas as perguntas apresentadas.

Dentro do “universo do front-end”, o profissional deve escolher a(s) área(s) que mais tem afinidade/facilidade e rumar nesse caminho.

Back-End

Definição curta: Back-end cuida do processamento dos dados. Acesso às bases de dados, servidores de e-mails e etc.

Sempre que alguém acessa um site, “algo” está acontecendo sem que a pessoa perceba. A “vida” de um site não é somente o que é apresentado no momento do acesso, já que existe uma gama de tecnologias e áreas de especialização que fazem com que a “mágica web” aconteça.

Na definição da Wikipédia, vista acima, consta que o front-end é uma espécie de interface entre o usuário e o back-end. Isso está mais ou menos correto. Realmente, no caso de um formulário de contato num site, por exemplo, toda a interface apresentada ao visitante foi planejada e desenvolvida pelas pessoas do front-end . Ao submeter aquele formulário, uma série de dados é enviada para o back-end para tratamento.

Mas, antes mesmo de aquele formulário aparecer, muito do trabalho do back-end já entrou em ação. Por exemplo, o formulário aparece corretamente ao visitante, pois está bem codificado em HTML. Mas não é raro que a própria marcação HTML seja “gerada” através de alguma linguagem de programação. No back-end não há somente a preocupação em tratar os dados que são enviados através da interface de um site ou sistema online, mas, muitas vezes, também de gerar a própria marcação, que gera o próprio formulário, para que os dados possam ser fornecidos.

Mas é no back-end, principalmente, que está presente e acontece toda a estrutura tecnológica que garante que as requisições aos sites funcionem; que a organização da estrutura dos mesmos seja feita de forma mais fácil; que possa haver o processamento das requisições e dados enviados ao servidor; que aconteça o correto armazenamento de dados em bases de dados. Enfim, é no back-end que está a lógica da aplicação e acontece o processamento dos dados para que se tornem informações.

Algumas das tecnologias pertencentes ao back-end são:

Linguagens de programação:

  • PHP (que você está estudando)
  • Ruby
  • Python
  • Java
  • C#
  • Etc.
Banco de dados:
  • MySQL
  • Oracle
  • PostgreSQL
  • MongoDB
  • Etc.

Estas são algumas das perguntas que cabe ao profissional do back-end responder, e ser tecnicamente capaz de solucionar/desenvolver:

  • “Depois que o formulário for submetido, o que acontece?”
  • “Eu terei acesso aos dados dos clientes de qual maneira?”
  • “Preciso que o cliente consiga exportar seus dados para uma planilha. É possível?”

O trabalho no back-end é de suma importância para que qualquer projeto no ambiente online possa começar a funcionar. Sem as tecnologias e os profissionais da área de desenvolvimento web (que é maior do que os exemplos dados), não é possível que a própria web exista! É disso que este curso trata. O foco será no back-end e você aprenderá PHP, uma linguagem que roda no lado do servidor.
